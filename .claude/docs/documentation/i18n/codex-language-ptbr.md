# Codex: Guide for Translating to Brazilian Portuguese

> **Prefix:** `codex-` | **Type:** Reference Manual | **Scope:** Conventions and guidance for technical translation to pt-BR

## Content

### Technical Writing Conventions in pt-BR

1. **Active voice** is preferred over passive voice
2. **Direct sentences** — avoid roundabout and ornate constructions
3. **Short paragraphs** — facilitate on-screen reading
4. **Terminological consistency** — use the same term for the same concept throughout

### Formalization Patterns

| Context | Correct pattern | Incorrect pattern |
|---------|-----------------|-------------------|
| Obligation | "O agente **DEVE**..." | "O agente tem que..." |
| Necessity | "É necessário..." | "Precisa..." |
| Recommendation | "Recomenda-se..." | "É bom que..." |
| Direct instruction | "Crie o arquivo..." | "Você poderia criar o arquivo..." |

### Common Pitfalls

1. **"Deletar" vs "Excluir"** — Prefer "excluir" in formal context
2. **"Setar" vs "Configurar"** — Prefer "configurar"
3. **"Rodar" vs "Executar"** — Prefer "executar" in formal context
4. **"Checar" vs "Verificar"** — Prefer "verificar"
5. **Excessive gerund** — Avoid "Estando o agente configurando..." → "Ao configurar, o agente..."

# Codex: Guide for Translating to Spanish

> **Prefix:** `codex-` | **Type:** Reference Manual | **Scope:** Conventions and guidance for technical translation to Spanish (es)

## Overview

This Codex is the practical guide for translating technical documentation **to neutral Spanish (es)**. It complements `lex-language-es` (mandatory rules) with guidance, examples, and reference tables.

## Context

- **Domain:** Technical translation to Spanish
- **Audience:** `warrior-translator` and agents translating to Spanish
- **Updates:** whenever new Spanish conventions are identified

## Content

### Technical Writing Conventions in Spanish

1. **Impersonal voice** is preferred: "Se debe configurar..." instead of "Debes configurar..."
2. **Clear, direct sentences** — avoid complex constructions
3. **Register consistency** — maintain the same formality level throughout
4. **Opening marks** — always use ¿ and ¡ at the start of questions and exclamations

### Formalization Patterns

| Context | Correct pattern | Incorrect pattern |
|---------|-----------------|-------------------|
| Obligation | "Se **DEBE** configurar..." | "Tienes que configurar..." |
| Necessity | "Es necesario..." | "Hace falta..." (colloquial) |
| Recommendation | "Se recomienda..." | "Estaría bien que..." |
| Prohibition | "**NO SE DEBE**..." | "No hay que..." |

### Common Pitfalls from pt-BR

1. **Maintaining pt-BR structure** — Spanish has different word order in some contexts
2. **False cognates** — The table above is essential
3. **Punctuation** — Do not forget opening ¿ and ¡ marks
4. **Gerund** — Spanish uses gerunds less than pt-BR

### Common Pitfalls from en

1. **Excessive passive voice** — English uses more passive voice; in Spanish, prefer active or impersonal voice
2. **"Actual"** (en = real) ≠ "actual" (es = current)
3. **"Sensible"** (en = wise) ≠ "sensible" (es = sensitive)
4. **"Eventually"** (en = finally) ≠ "eventualmente" (es = possibly)
